<Section id="http-only" num="03 — Transport" title="HTTPS — what it enables">

HTTPS went live 2026-06-03. The constraints that gated us on HTTP are now removed:

| Was | Now |
|---|---|
| PayFast ITN delivery unreliable on some networks | PayFast ITN over HTTPS lands reliably; pull-reconciliation remains as a belt-and-braces fallback |
| Browser security warnings on the operator UI | Real Let's Encrypt cert, green padlock, no warnings |
| Third-party HTTPS-only callbacks couldn't reach us | Now accept HTTPS-only callback URLs |
| CareFirst-side webhook would land over HTTP | **Unblocked** — see <a href="/reports/consultation-outcome-webhook">Consultation Outcome Webhook</a> |

</Section>

<Section id="https-plan" num="04 — HTTPS rollout" title="How HTTPS was set up">

Shipped 2026-06-03:

1. **Subdomain provisioned:** IT created the `bookings.carefirst.co.za` A-record pointing at `187.127.135.11`.
2. **Traefik configured for HTTP-01 ACME** — `--certificatesresolvers.letsencrypt.acme.httpchallenge=true` + entrypoints `web:80` and `websecure:443`. Cert storage at `/letsencrypt/acme.json`, ACME email in `/docker/traefik/.env`.
3. **Booking-app docker-compose labels** point a router at the `websecure` entrypoint with `Host(\`bookings.carefirst.co.za\`)` + `tls.certresolver=letsencrypt`.
4. **Global HTTP→HTTPS redirect** at the Traefik entrypoint level so any bare-host hit upgrades automatically.
5. **`NEXT_PUBLIC_APP_URL` = `https://bookings.carefirst.co.za`** in the Hostinger Docker Manager Environment panel.

Side bonus: the FortiGuard "Unrated IP" block at the First Care office is resolved — `carefirst.co.za` is rated as Business so the subdomain passes through.

</Section>

<Section id="deploy" num="05 — Deploy" title="Deploy and rollback">

We deploy by SSHing to the VPS and running the deploy one-liner. Every deploy tags the new image with the commit SHA, keeping the previous image available locally as a rollback target.

```bash
cd /opt/3rd-Party-Booking-System && \
  git pull origin main && \
  export IMAGE_TAG=$(git rev-parse --short HEAD) && \
  cd booking-app && \
  docker compose build && \
  docker compose up -d
```

Rollback is a single command — no rebuild, ~10 seconds:

```bash
export IMAGE_TAG=<previous-short-sha>
docker compose up -d --no-build
```

Health check post-deploy:

```bash
curl -sf http://127.0.0.1:3000/api/health
# → { "status": "ok", "time": "...", "checks": { "db": "ok" } }
# or externally:
curl -sf https://bookings.carefirst.co.za/api/health
```

</Section>

<Section id="implications" num="06 — Implications" title="Implications for CareFirst">

<Callout title="Inbound HTTPS webhooks are unblocked">
HTTPS shipped 2026-06-03. CareFirst can now build the consultation-outcome webhook directly against <code>https://bookings.carefirst.co.za</code> with shared-secret + signature validation. Both sides can move forward on the new states (<code>Consultation Complete</code>, <code>No-show</code>, etc.).
</Callout>

<Callout title="Outbound HTTPS unchanged">
Our outbound SSO auto-register call and PayFast / Supabase / SMTP calls were always HTTPS. The change is on the inbound side — patient traffic, PayFast ITN, and now CareFirst webhooks all land over TLS.
</Callout>

</Section>
